# 🛠 Правила работы с репозиторием (Git Flow)

Этот документ описывает стандартный процесс работы над проектом. Соблюдение этих правил гарантирует, что мы не сломаем код друг друга и проект всегда будет в рабочем состоянии.

## ⛔ ЗОЛОТОЕ ПРАВИЛО
**НИКТО И НИКОГДА не пушит код напрямую в ветку `main`.**
Ветка `main` — это "продакшен". Туда попадает только полностью рабочий код через Pull Request (Merge Request).

---

## 📦 БЛОК 1: Правила именования (Ветки и Коммиты)

Все названия веток и коммитов пишем **на английском языке**.

### 1. Названия веток
Формат: `тип/краткое-описание-задачи`
* `feat/...` — для новой фичи (Пример: `feat/ue5-grid-setup`, `feat/agent-movement`)
* `fix/...` — для исправления бага (Пример: `fix/python-import-error`)
* `docs/...` — для документации/ридми (Пример: `docs/update-readme`)
* `refactor/...` — переписывание кода без добавления новых фич (Пример: `refactor/agent-class-structure`)

### 2. Сообщения коммитов (Conventional Commits)
Формат: `тип: описание того что сделано (с маленькой буквы)`
* `feat: add base agent class with coordinates`
* `fix: prevent agent from moving out of bounds`
* `chore: update gitignore for unreal engine`
* `docs: add contributing guidelines`

---

## 🚀 БЛОК 2: Рабочий процесс (Шаг за шагом)

Каждый раз перед началом работы нужно забрать изменения, которые сделали другие, и создать свою ветку.

**Сценарий работы в терминале:**

```bash

git clone https://git.wmi.amu.edu.pl/s498817/ai-course-project.git
cd ai-course-project

# 1. Переходим в главную ветку
git checkout main

# 2. Скачиваем последние обновления
git pull

# 3. Создаем свою ветку для новой задачи и переходим в нее
git checkout -b feat/название-вашей-задачи

# ... ПИШЕМ КОД ...

# 4. Проверяем, какие файлы изменились
git status

# 5. Добавляем измененные файлы
git add .

# 6. Делаем коммит (сохраняем локально)
git commit -m "feat: short description of what you did"

# 7. Отправляем вашу ветку на сервер (в Gitea)
git push origin feat/название-вашей-задачи
```
**Шаг 8:** Заходим в Gitea, нажимаем **"New Pull Request"** из вашей ветки в `main`. Просим тиммейта проверить код и нажать Merge.

## 🆘 БЛОК 3: Решение частых проблем

**Ситуация 1: Я случайно начал писать код в ветке `main`!**
Ничего не коммитьте. Просто создайте новую ветку, Git сам перенесет ваши изменения в нее:
```bash
git checkout -b feat/my-new-code
```

**Ситуация 2: Git пишет `CONFLICT` при слиянии веток.**
Вы с тиммейтом изменили одну и ту же строчку. Откройте файл с ошибкой в IDE (VS Code/PyCharm). Выберите нужный вариант кода, затем:
```bash
git add .
git commit -m "fix: resolve merge conflicts"
```

**Ситуация 3: Я хочу вернуть файлы к состоянию последнего коммита.**
*Осторожно, это удалит все несохраненные изменения!*
```bash
git restore .
```

**Ситуация 4 (Для Unreal Engine): Git не дает сделать push, пишет "File is too large" (>100MB).**
Вы случайно добавили тяжелый ассет. Отмените последний коммит (файлы останутся на диске):
```bash
git reset HEAD~1
```
После этого удалите тяжелый файл или добавьте его в `.gitignore`, и заново сделайте `add` и `commit`.

## 🗄️ Работа с тяжелыми ассетами (DVC - Data Version Control)

Unreal Engine 5 использует тяжелые файлы (текстуры, 3D модели, `.uasset`), которые нельзя хранить в обычном Git, чтобы не перегрузить сервер университета. Для этого мы используем индустриальный стандарт **DVC**, который связывает Git с нашим Google Drive.

### 🚫 Важное правило для Unreal Engine
**ВРЕМЕННЫЕ ФАЙЛЫ НЕЛЬЗЯ ПУШИТЬ НИ В GIT, НИ В DVC!**
Никогда не добавляйте папки `Saved`, `Intermediate`, `DerivedDataCache`, `Binaries` и `Build`. Они должны игнорироваться локально. DVC используется ТОЛЬКО для полезных ассетов (папка `Content`).

---

### 📥 1. Как получить тяжелые файлы (Для Саши и Никиты)
Если Артем добавил новые текстуры в UE5, они не скачаются через обычный `git pull`. В Git прилетят только крошечные файлы-указатели `.dvc`.

Чтобы скачать сами 3D-модели и текстуры:
1. Убедитесь, что у вас установлен DVC: `pip install dvc dvc-gdrive`
2. Обновите код из Git: `git pull origin main`
3. Скачайте тяжелые файлы с Google Drive:
   ```bash
   dvc pull
   ```
*(При первом запуске откроется браузер для авторизации в Google).*

---

### 📤 2. Как добавить новые тяжелые файлы (Для Артема - Frontend)
Если ты добавил новую модельку или текстуру в папку `frontend_ue/Content/`, делай следующее:

1. **Добавь файл в DVC:**
   ```bash
   dvc add frontend_ue/Content/МояТекстура.png
   ```
   *(DVC создаст файл `МояТекстура.png.dvc` и спрячет саму картинку).*

2. **Зафиксируй указатель в Git:**
   ```bash
   git add frontend_ue/Content/МояТекстура.png.dvc
   git commit -m "feat: add heavy texture for Mars surface"
   ```

3. **Отправь данные на серверы:**
   ```bash
   # Отправляем текстовый указатель преподавателю в Gitea
   git push origin feat/название-твоей-ветки
   
   # Отправляем саму тяжелую картинку на Google Drive
   dvc push
   ```

### 🚑 3. Частые ошибки и их решение (Troubleshooting)

*   **Ошибка:** `ERROR: failed to pull data from the cloud - unexpected error`
    **Решение:** Срок действия токена Google Drive истек. Напиши в терминале `dvc remote modify myremote gdrive_use_service_account false`, затем снова `dvc pull` и пройди авторизацию в браузере.
*   **Ошибка (в UE5):** Unreal Engine жалуется на "Corrupted data" или не видит текстуры после `git pull`.
    **Решение:** Ты забыл сделать `dvc pull`. Unreal Engine пытается прочитать текстовый файл-указатель `.dvc` вместо самой картинки. Напиши `dvc pull` и перезапусти UE5.
*   **Вопрос:** "Я случайно сделал `git add` тяжелого файла вместо `dvc add`!"
    **Решение:** Не делай commit! Напиши `git reset HEAD <имя_файла>`, чтобы убрать его из Git, и затем используй `dvc add`.
